# Security Architecture Overview

---

# Overview

This document provides a **high-level overview of the Microsoft 365 security architecture** implemented within the environment.

The architecture is designed using **Zero Trust security principles**, where access to organisational resources is continuously verified based on identity, device posture, and contextual security signals.

Rather than relying on traditional network-based security boundaries, this architecture focuses on **identity-driven access control**, ensuring that every authentication request is evaluated before access to services is granted.

The architecture integrates multiple Microsoft security platforms to provide layered protection across **identities, devices, applications, and data**.

---

# Core Security Platforms

The Microsoft 365 security architecture integrates several key Microsoft security services that work together to protect organisational resources.

---

### Microsoft Entra ID

**Microsoft Entra ID** functions as the central identity provider for the organisation.

Responsibilities include:

* **User authentication and identity verification**
* **Role-Based Access Control (RBAC)**
* **Conditional Access policy enforcement**
* **Identity risk detection**
* **Token issuance for Microsoft 365 workloads**

All authentication requests are processed through Microsoft Entra ID before users can access Microsoft 365 services.

---

### Microsoft Intune

**Microsoft Intune** provides endpoint management and device compliance enforcement.

Device security signals generated by Intune are used by **Conditional Access policies** to determine whether devices meet organisational security requirements.

Capabilities include:

* **Device enrollment and management**
* **Device compliance evaluation**
* **Security configuration enforcement**
* **Mobile device and application management**

This ensures that only **trusted and compliant devices** can access organisational resources.

---

### Microsoft Defender Security Platform

The **Microsoft Defender security ecosystem** provides threat detection and monitoring across the environment.

Key Defender services include:

* **Microsoft Defender for Endpoint** – Detects and responds to endpoint threats
* **Microsoft Defender for Identity** – Monitors identity-based attacks and suspicious authentication behaviour
* **Microsoft Defender for Office 365** – Protects against email-based threats such as phishing and malware
* **Microsoft Defender for Cloud Apps** – Monitors cloud application usage and identifies risky behaviour

These tools provide **advanced threat detection across identities, endpoints, and applications**.

---

### Microsoft Purview

**Microsoft Purview** provides data protection and compliance capabilities across the Microsoft 365 environment.

Capabilities include:

* **Data classification and sensitivity labels**
* **Data Loss Prevention (DLP) policies**
* **Insider Risk Management**
* **eDiscovery and compliance investigations**

Purview ensures that **sensitive organisational data remains protected regardless of where it is accessed or shared**.

---

# Security Architecture Layers

The architecture is structured in multiple security layers that work together to protect organisational resources.

---

### Identity Layer

The identity layer is responsible for verifying user identities before access to services is granted.

Key components include:

* **Microsoft Entra ID authentication**
* **Multi-Factor Authentication (MFA)**
* **Identity Protection risk evaluation**

Identity verification acts as the **first line of defence** in the architecture.

---

### Device Security Layer

The device layer ensures that devices accessing organisational resources meet defined security requirements.

This layer relies on **Microsoft Intune device compliance policies**.

Device evaluation may include:

* Device encryption status
* Operating system updates
* Endpoint protection configuration
* Device enrollment status

Only compliant devices are permitted to access sensitive services.

---

### Access Control Layer

Access control is enforced through **Conditional Access policies** within Microsoft Entra ID.

These policies evaluate multiple authentication signals before granting access.

Examples include:

* Requiring **MFA for administrative accounts**
* Restricting access from **untrusted locations**
* Enforcing **device compliance requirements**
* Blocking **high-risk authentication attempts**

This ensures that access decisions are **context-aware and dynamically enforced**.

---

### Threat Protection Layer

Threat detection capabilities are provided through **Microsoft Defender services**.

These tools continuously monitor the environment for indicators of compromise such as:

* Suspicious login activity
* Endpoint malware infections
* Phishing attacks
* Risky cloud application usage

Security alerts generated by Defender help security teams quickly detect and respond to threats.

---

### Data Protection Layer

The data protection layer ensures that organisational data remains protected even after access is granted.

Using **Microsoft Purview**, organisations can implement:

* **Sensitivity labels**
* **Data Loss Prevention policies**
* **Compliance and investigation tools**

This layer ensures that sensitive information remains protected across Microsoft 365 services.

---

# Microsoft 365 Workloads Protected

The architecture secures access to several core Microsoft 365 workloads including:

* **Exchange Online**
* **SharePoint Online**
* **Microsoft Teams**
* **OneDrive for Business**

Access to these services is governed through **identity authentication, device compliance validation, and Conditional Access enforcement**.

---

# Zero Trust Security Model

The architecture is based on Microsoft's **Zero Trust security framework**.

Key principles include:

---

### Verify Explicitly

Every authentication request is evaluated using multiple security signals including identity, device posture, and risk indicators.

---

### Least Privilege Access

Users are granted the **minimum level of access required** to perform their responsibilities.

---

### Assume Breach

Security monitoring tools continuously observe authentication behaviour and system activity to detect potential threats.

---

# Summary

This Microsoft 365 security architecture integrates **identity protection, device security, threat detection, and data protection** into a unified security framework.

By combining **Microsoft Entra ID, Microsoft Intune, Microsoft Defender, and Microsoft Purview**, the organisation establishes a layered security architecture aligned with **Zero Trust principles**.

This approach significantly strengthens the organisation’s ability to **protect identities, secure devices, detect threats, and safeguard sensitive organisational data**.
