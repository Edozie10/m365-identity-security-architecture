Identity Governance Design

Overview

Identity governance ensures that users have the appropriate level of access required to perform their roles while minimizing security risks.

This project implements Role Based Access Control (RBAC) within Microsoft Entra ID.

Role Assignments

Example roles used in this environment:

Global Administrator
Responsible for full administrative access to the Microsoft 365 tenant.

User Administrator
Responsible for managing users and group memberships.

Security Administrator
Responsible for monitoring security configurations and enforcing policies.

 Group Management

Users are organized into groups to simplify access management.

Example groups:

- HR Team
- Engineering Team
- IT Administrators

Group membership allows administrators to efficiently assign permissions to multiple users.
