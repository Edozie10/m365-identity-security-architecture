# Monitoring and Security Operations

---

# Overview

Security architecture is only effective when supported by continuous monitoring and operational response capabilities.

Within this environment, monitoring and security operations are performed using telemetry and alerts generated across Microsoft 365 security services.

Security monitoring integrates signals from:

* **Microsoft Entra ID**
* **Microsoft Defender**
* **Microsoft Intune**
* **Microsoft Purview**

These tools provide visibility into authentication behaviour, device health, and potential security incidents across the organisation.

---

# Identity Monitoring

---

### Microsoft Entra ID Sign-In Logs

Microsoft Entra ID provides detailed authentication logs that allow administrators to monitor identity activity.

Security teams can review:

* **Successful sign-in attempts**
* **Failed authentication attempts**
* **Unusual login locations**
* **Conditional Access policy outcomes**
* **Multi-Factor Authentication activity**

Monitoring authentication logs helps identify suspicious login behaviour and potential identity compromise.

---

### Identity Risk Detection

Microsoft Entra ID **Identity Protection** identifies potentially risky authentication events.

Examples of risk indicators include:

* **Impossible travel sign-ins**
* **Unusual authentication patterns**
* **Credentials exposed in breach databases**
* **Suspicious login behaviour**

Conditional Access policies may automatically require additional verification when risk signals are detected.

---

# Endpoint Monitoring

---

### Microsoft Defender for Endpoint

Microsoft Defender for Endpoint provides endpoint detection and response capabilities.

Security teams can monitor endpoints for:

* **Malware infections**
* **Suspicious application behaviour**
* **Unauthorized system changes**
* **Exploitation of vulnerabilities**

Security alerts generated by Defender help administrators investigate potential security incidents.

---

### Device Compliance Monitoring

Microsoft Intune provides reporting on device compliance and device security posture.

Administrators can monitor:

* **Device compliance status**
* **Device enrollment status**
* **Operating system security updates**
* **Endpoint protection configuration**

Devices that fall out of compliance may lose access to organisational resources through Conditional Access enforcement.

---

# Email Security Monitoring

---

### Microsoft Defender for Office 365

Microsoft Defender for Office 365 protects organisational email and collaboration services.

Security monitoring includes detection of:

* **Phishing attacks**
* **Malicious attachments**
* **Suspicious email links**
* **Business email compromise attempts**

Administrators can investigate alerts and remove malicious messages from user mailboxes.

---

# Cloud Application Monitoring

---

### Microsoft Defender for Cloud Apps

Microsoft Defender for Cloud Apps monitors how users interact with cloud services.

Capabilities include:

* **Detecting risky user behaviour**
* **Identifying unsanctioned applications (Shadow IT)**
* **Investigating suspicious user sessions**

This helps organisations maintain visibility into cloud application usage.

---

# Data Security Monitoring

---

### Microsoft Purview Alerts

Microsoft Purview generates alerts related to data protection policies.

Examples include:

* **Data Loss Prevention violations**
* **Sensitive data sharing events**
* **Insider risk activity**

These alerts allow administrators to investigate potential data security incidents.

---

# Security Incident Response

When suspicious activity is detected, security teams follow structured response procedures.

Typical response actions include:

* **Investigating authentication logs**
* **Reviewing security alerts**
* **Resetting compromised user credentials**
* **Isolating affected devices**
* **Updating security policies if necessary**

Security teams rely on telemetry from **Entra ID, Defender, Intune, and Purview** to determine the root cause of incidents.

---

# Continuous Security Improvement

Monitoring insights allow organisations to refine security policies and improve protection over time.

Security teams regularly review:

* **Authentication patterns**
* **Security alert trends**
* **Device compliance metrics**
* **Data protection alerts**

These insights help strengthen the organisation’s security posture.

---

# Summary

Monitoring and security operations provide the visibility required to maintain a secure Microsoft 365 environment.

By leveraging telemetry from **Microsoft Entra ID, Microsoft Defender, Microsoft Intune, and Microsoft Purview**, security teams can detect threats, investigate incidents, and continuously improve the organisation’s security posture.
