 Architecture Design

Overview

This architecture demonstrates how Microsoft Entra ID acts as the central identity provider for a Microsoft 365 environment.

Users authenticate through Microsoft Entra ID before accessing organizational resources.

 Components

 Users

Example users in the environment:

- Admin User
- HR User
- Engineering User

 Identity Provider

Microsoft Entra ID provides:

- Identity authentication
- Role-based access control
- Conditional access enforcement

 Security Controls

Security controls implemented include:

- Multi-Factor Authentication (MFA)
- Conditional Access policies
- Identity governance using RBAC

Microsoft 365 Services

Users gain access to services including:

- Exchange Online
- SharePoint Online
- Microsoft Teams

Access to these services is controlled through identity and access policies configured in Microsoft Entra ID.
